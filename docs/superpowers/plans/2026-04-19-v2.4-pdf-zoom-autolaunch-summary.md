# v2.4 PDF 缩放 / 开机自启 / 阅读器摘要 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship three independent upgrades for Shibei v2.4: PDF zoom (toolbar + shortcuts + per-resource persistence), system auto-launch (minimized, macOS + Windows), and summary display in the reader's annotation panel.

**Architecture:** Three vertical slices sharing only i18n scaffolding.
- **PDF zoom** — introduce a `zoomFactor` state in `PDFReader`; render pages inside a new `.content` wrapper whose width = `zoomFactor × container`; scale flows into canvas/TextLayer/highlight rects via the existing formula. Persist per-resource in `sessionState.ReaderTab.pdfZoom`.
- **Auto-launch** — `tauri-plugin-autostart` registers the app with the OS; `--autostart` arg triggers `window.minimize()` during Tauri setup; Settings → Appearance gains a toggle.
- **Summary** — move `AnnotationsHeader` into `scrollArea`; add a `SummarySection` above it using existing `cmd.getResourceSummary`; add a top-side sticky mirror of `stickyNotesHeader`.

**Tech Stack:** Rust (Tauri 2.x, `tauri-plugin-autostart`), TypeScript/React 19 (`pdfjs-dist`, `react-i18next`), Vitest.

Create `src/lib/pdfZoom.ts`:

```typescript
export const ZOOM_MIN = 0.5;
export const ZOOM_MAX = 4.0;
export const ZOOM_STEP = 0.05;
export const ZOOM_DEFAULT = 1.0;

export interface PdfPageInfo {
  width: number;
  height: number;
}

function round2(n: number): number {
  return Math.round(n * 100) / 100;
}

export function clampZoom(value: number): number {
  if (!Number.isFinite(value)) return ZOOM_DEFAULT;
  if (value < ZOOM_MIN) return ZOOM_MIN;
  if (value > ZOOM_MAX) return ZOOM_MAX;
  return value;
}

export function nextZoom(current: number): number {
  return round2(Math.min(ZOOM_MAX, current + ZOOM_STEP));
}

export function prevZoom(current: number): number {
  return round2(Math.max(ZOOM_MIN, current - ZOOM_STEP));
}

export function computePageHeights(
  infos: PdfPageInfo[],
  containerWidth: number,
  zoom: number,
): number[] {
  if (!containerWidth || infos.length === 0) return [];
  const effectiveWidth = containerWidth * zoom;
  return infos.map((info) => info.height * (effectiveWidth / info.width));
}
```

const prevZoomRef = useRef<number>(zoomFactor);
useEffect(() => {
  const container = containerRef.current;
  if (!container || pageInfos.length === 0) return;
  if (prevZoomRef.current === zoomFactor) return;

  // Preserve vertical position via fraction (same trick as ResizeObserver)
  const fraction = container.scrollHeight > 0
    ? container.scrollTop / container.scrollHeight
    : 0;

  renderGenRef.current += 1;
  renderedPagesRef.current.clear();

  // Re-render next frame after layout applies the new wrapper width

      container.scrollLeft = (container.scrollWidth - container.clientWidth) / 2;
    } else {
      container.scrollLeft = 0;
    }
    renderVisiblePages();
  });

  prevZoomRef.current = zoomFactor;

- [ ] **Step 1: Add keyboard handler hook**

In `ReaderView.tsx`, after `handleZoomReset`, add a `useEffect` that listens on `window`:

```typescript
// Keyboard shortcuts for PDF zoom (Ctrl/Cmd +/-/0)
useEffect(() => {
  if (resource.resource_type !== "pdf") return;

  const isEditableTarget = (el: EventTarget | null): boolean => {
    if (!(el instanceof HTMLElement)) return false;
    const tag = el.tagName;
    if (tag === "INPUT" || tag === "TEXTAREA") return true;
    if (el.isContentEditable) return true;
    return false;
  };

  const handler = (e: KeyboardEvent) => {
    if (!(e.ctrlKey || e.metaKey)) return;
    if (isEditableTarget(e.target)) return;
    if (e.key === "+" || e.key === "=") {
      e.preventDefault();
      handleZoomIn();
    } else if (e.key === "-") {
      e.preventDefault();
      handleZoomOut();
    } else if (e.key === "0") {
      e.preventDefault();
      handleZoomReset();
    }
  };

  window.addEventListener("keydown", handler);
  return () => window.removeEventListener("keydown", handler);
}, [resource.resource_type, handleZoomIn, handleZoomOut, handleZoomReset]);
```

Note: only the **active** reader tab's `ReaderView` is mounted (per the lazy-mount pattern in `App.tsx`), so we don't need to check "is active tab".
