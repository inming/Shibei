# v1.6 — Markdown 标注设计文档

## 目标

评论和资料级笔记支持 Markdown 格式，提升标注表达能力。纯前端改造，后端零改动。

## 决策记录

| 决策点 | 结论 | 理由 |
|-------|------|------|
| 编辑模式 | textarea + 切换预览 | 评论通常很短，所见即所得编辑器过重；分屏在侧栏空间不够 |
| 渲染库 | react-markdown + remark-gfm | React 组件式渲染，天然防 XSS，无需 DOMPurify；GFM 支持表格/删除线/任务列表 |
| 代码高亮 | 不引入 | 资料标注非代码笔记场景，等宽字体 + 灰底足够 |
| 数据层 | 零改动 | 纯文本即合法 Markdown，旧数据自动兼容 |
| 搜索索引 | 原样索引 | trigram 分词下语法符号不影响召回 |
| 搜索高亮 | react-markdown 自定义 text 组件 | 在渲染后的文本叶节点上做匹配，用户所见即所搜 |
| 后端 | 零改动 | Markdown 是纯展示层逻辑 |

## 新增依赖

| 包名 | 用途 | 理由 |
|------|------|------|
| `react-markdown` | Markdown → React 元素渲染 | AST 转 React 组件，不用 dangerouslySetInnerHTML，天然安全 |
| `remark-gfm` | GFM 扩展 | 表格、删除线、任务列表、自动链接 |

不引入：DOMPurify、语法高亮库、富文本编辑器。

## 数据层

**数据库不变。** `comments.content` 字段继续存 TEXT，不加 `content_format` 字段。所有 content 统一按 Markdown 渲染——纯文本是合法 Markdown，渲染结果与当前一致。

**搜索索引不变。** FTS5 继续索引原始 content（含 Markdown 语法符号），trigram 分词下 `**重要**` 搜"重要"完全命中。

**同步不变。** 后端 CRUD、sync_log、HLC 全部不受影响。

## 编辑交互

### 现有行为保持

- `<textarea>` 输入，auto-resize（min 40px, max 200px）
- Enter 提交，Shift+Enter 换行，Escape 取消
- 新建/编辑评论默认编辑态

### 新增：预览切换

textarea 区域右上角增加切换按钮：

- **编辑态**（默认）：显示 textarea，按钮文字"预览"
- **预览态**：隐藏 textarea，同位置显示 react-markdown 渲染结果，按钮文字"编辑"
- 预览态下 Enter/Escape 快捷键不生效
- 切换不丢失内容（textarea value 保持）

### 应用范围

| 位置 | 编辑（textarea + 预览切换） | 展示（react-markdown 渲染） |

| AnnotationPanel — highlight 评论输入 | Yes | — |
| AnnotationPanel — highlight 评论编辑 | Yes | — |
| AnnotationPanel — 评论展示 | — | Yes |

## Markdown 渲染

### 支持的语法

标题、粗体、斜体、删除线、有序/无序列表、任务列表、行内代码、代码块、链接、引用块、表格。

### 不支持的语法

- **图片**：标注场景无意义。react-markdown 通过 `components.img` 替换为纯链接文本。
- **HTML 内嵌**：react-markdown 默认不渲染 raw HTML，保持安全。

### 渲染组件

```tsx
import ReactMarkdown from "react-markdown";
import remarkGfm from "remark-gfm";

function MarkdownContent({ content }: { content: string }) {
  return (
    <ReactMarkdown
      remarkPlugins={[remarkGfm]}
      components={{
        img: ({ src, alt }) => <a href={src}>{alt || src}</a>,
        a: ({ href, children }) => (
          <a href={href} target="_blank" rel="noopener noreferrer">
            {children}
          </a>
        ),
      }}
    >
      {content}
    </ReactMarkdown>
  );
}
```

抽取为公共组件 `MarkdownContent`，AnnotationPanel 和 PreviewPanel 共用。

## 搜索高亮

### 现有机制

PreviewPanel 的 `highlightMatch(text, query)` 对纯文本做字符串匹配，命中部分包在 `<mark>` 里。

### 改造方式

利用 react-markdown 的 `components` prop，当存在 searchQuery 时，自定义 `text` 节点渲染：

```tsx
function MarkdownContent({ content, searchQuery }: {
  content: string;
  searchQuery?: string;
}) {
  const components = searchQuery
    ? {
        text: ({ children }: { children: string }) =>
          highlightMatch(String(children), searchQuery),
        img: /* ... */,
        a: /* ... */,
      }
    : {
        img: /* ... */,
        a: /* ... */,
      };

  return (
    <ReactMarkdown remarkPlugins={[remarkGfm]} components={components}>
      {content}
    </ReactMarkdown>
  );
}
```

无 searchQuery 时不传 text 覆盖，零开销。

## 样式设计

### 容器

MarkdownContent 渲染结果包裹在 `.markdownContent` class 内，用后代选择器定义子元素样式。AnnotationPanel 和 PreviewPanel 各自的 CSS Module 中定义，保持样式隔离。

### 元素样式

| 元素 | 样式要点 |
|------|---------|
| `h1-h6` | 缩小字号（h1 ≈ 正文 × 1.1），标注区空间有限 |
| `p` | 行高 1.4，段间距 0.4em |
| `ul/ol` | 左缩进 1.2em，紧凑间距 |
| `code`（行内） | `var(--color-code-bg)` 背景 + 等宽字体 + 2px 圆角 |
| `pre > code`（代码块） | 块级灰底 + 横向滚动 + 等宽字体 + padding |
| `blockquote` | 左 3px 边框 `var(--color-accent)` + 浅灰背景 |
| `a` | `var(--color-accent)` 颜色 + 下划线，点击行为同外链（可点击，在系统浏览器打开） |
| `table` | 1px 边框 + 紧凑 padding + 横向滚动 |
| `del` | 删除线 |
| `hr` | 1px 分隔线 |
| `input[type=checkbox]` | GFM 任务列表复选框，只读 |

### 深色模式

新增 CSS 变量（在 `variables.css` 中同时定义 light/dark）：

| 变量名 | Light 值 | Dark 值 | 用途 |
|-------|----------|---------|------|
| `--color-code-bg` | `#f5f5f5` | `#2a2a2a` | 行内代码 + 代码块背景 |
| `--color-blockquote-bg` | `#f9f9f9` | `#1e1e1e` | 引用块背景 |
| `--color-table-border` | `#e0e0e0` | `#3a3a3a` | 表格边框 |

### 预览切换按钮

小型文字按钮，风格与现有"编辑"/"删除"按钮一致：
- 字号 `var(--font-size-xs)`
- 颜色 `var(--color-text-secondary)`，hover 时 `var(--color-accent)`
- 置于 textarea 容器右上角
