# v1.8 — 多语言 (i18n) 设计文档

## 目标

UI 文案国际化，支持中英文切换。默认中文，用户可在设置页切换语言。

## 范围概述

### 文案盘点

| 区域 | 文件数 | 大约字符串数 | 说明 |
|------|--------|-------------|------|
| React 组件 | 25 | ~370 | 按钮、标签、占位符、确认弹窗、状态文本、空状态等 |
| React Hooks | 4 | ~13 | 同步状态消息、操作提示 |
| Chrome 插件 | 3 | ~55 | popup UI、选区脚本提示 |
| Rust 后端 | 4 | ~55 | 错误消息（经由 toast 展示给用户） |
| **合计** | **36** | **~490** | 去重后约 400 条唯一字符串 |

### 文案分类

| 类型 | 数量 | 示例 |
|------|------|------|
| 按钮/操作标签 | ~80 | 保存、取消、删除、编辑、复制链接 |
| 表单标签/占位符 | ~90 | "密码至少 8 个字符"、"搜索..." |
| 弹窗标题/区块标题 | ~50 | "确认删除"、"同步设置"、"端到端加密" |
| 成功/状态消息 | ~80 | "保存成功！"、"链接已复制"、"已同步" |
| 错误消息 | ~60 | "删除失败"、"连接失败"、"PIN 码不正确" |
| 确认消息 | ~30 | "确定删除该资料吗？"、"此操作无法撤销" |
| 帮助文本/提示 | ~40 | "数据以明文存储在 S3..."、"将加密密钥保存在系统钥匙串中..." |

### 含变量插值的字符串

部分字符串包含动态变量，需要参数化处理：
- `"确定删除此高亮标注？关联的 ${commentCount} 条评论也会一并删除"`
- `"该 URL 已保存过 ${count} 次"`
- `"已选 ${count} 项"`
- `"发现 ${count} 个孤儿文件（共 ${size} MB）"`
- `"最后同步: ${time}"`

中文无复数形态。英文使用 i18next 内置的 `_one`/`_other` 后缀约定处理复数，无额外成本：

```json
// en/annotation.json
{
  "itemsSelected_one": "{{count}} item selected",
  "itemsSelected_other": "{{count}} items selected"
}
```

调用 `t('itemsSelected', { count })` 自动选择正确形式。中文语言包不需要后缀，只写一个 key 即可。

## 决策记录

| 决策点 | 结论 | 理由 |
|-------|------|------|
| i18n 库 | `react-i18next` + `i18next` | 生态最成熟，文档最全，路线图已提及；桌面应用不敏感于 bundle 大小（22-56 kB gzipped）；团队学习成本最低 |
| 备选方案 | `@lingui/react`（10 kB）值得考虑 | 更轻量，自动提取强，但社区小、macro 语法有学习曲线 |
| 语言包格式 | JSON 文件（i18next 默认） | 简单直接，TypeScript 可类型检查 key |
| key 命名 | 命名空间 + 语义 key | 如 `sidebar.allResources`、`sync.saveConfig`，不用中文做 key |
| 命名空间 | 按功能模块拆分 | `common`、`sidebar`、`reader`、`annotation`、`settings`、`sync`、`encryption`、`lock`、`search`、`extension` |
| 后端错误消息 | 前端翻译 | 后端返回错误 code/key，前端查表翻译；不在 Rust 层做 i18n |
| Chrome 插件 | 独立语言包 | 插件用 Chrome i18n API（`chrome.i18n`，`_locales/` 目录），与主应用分开管理 |
| 默认语言 | 中文 (zh) | 当前所有文案为中文 |
| 语言持久化 | localStorage（key: `shibei-language`） | 与 theme 一致的持久化方式 |
| 语言检测 | 手动选择优先 → 系统语言回退 | `i18next-browser-languagedetector` 检测系统语言，用户选择后覆盖 |

### 方案对比（备忘）

| 库 | Bundle | TypeScript | 提取工具 | 社区 |
|----|--------|-----------|---------|------|
| react-i18next | 22-56 kB | 好（需配置） | 三方 scanner | 最大 |
| @lingui/react | ~10 kB | 优秀（自动生成） | 内置 extract CLI | 中 |
| typesafe-i18n | ~1.5 kB | 最佳（编译时） | 内置 generator | 小 |
| react-intl | ~18 kB | 好 | 内置 extract | 大 |
| 自定义 Context | ~0.5 kB | 手动维护 | 无 | 无 |

选 react-i18next 主要因为：桌面应用 bundle 不敏感、文档和示例最多、出问题最容易搜到解决方案。如果后续觉得太重，可以考虑迁移到 Lingui（API 风格相近）。

## 新增依赖

| 包名 | 用途 | 理由 |
|------|------|------|
| `i18next` | i18n 核心 | 语言包加载、插值、命名空间 |
| `react-i18next` | React 集成 | `useTranslation` hook、`<Trans>` 组件 |
| `i18next-browser-languagedetector` | 语言检测 | 检测系统语言作为默认值 |

不引入：i18next-http-backend（不需要远程加载语言包）、任何提取扫描工具（手动迁移，一次性工作）。

## 技术架构

### 目录结构

```
src/
  locales/
    zh/
      common.json        # 通用：确认、取消、删除、保存...
      sidebar.json       # 侧栏：全部资料、文件夹、标签、回收站
      reader.json        # 阅读器：反色阅读、加载中
      annotation.json    # 标注面板：标注、笔记、评论
      settings.json      # 设置页通用：外观
      sync.json          # 同步设置
      encryption.json    # 加密设置
      lock.json          # 锁屏
      search.json        # 搜索
    en/
      common.json
      sidebar.json
      reader.json
      annotation.json
      settings.json
      sync.json
      encryption.json
      lock.json
      search.json
  i18n.ts               # i18next 初始化配置
```

### 初始化

```typescript
// src/i18n.ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import LanguageDetector from 'i18next-browser-languagedetector';

// 静态导入所有语言包（桌面应用，无需懒加载）
import zhCommon from './locales/zh/common.json';
import zhSidebar from './locales/zh/sidebar.json';
// ... 其他命名空间
import enCommon from './locales/en/common.json';
import enSidebar from './locales/en/sidebar.json';
// ... 其他命名空间

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: {
      zh: { common: zhCommon, sidebar: zhSidebar, /* ... */ },
      en: { common: enCommon, sidebar: enSidebar, /* ... */ },
    },
    fallbackLng: 'zh',
    defaultNS: 'common',
    interpolation: { escapeValue: false }, // React 已防 XSS
    detection: {
      order: ['localStorage', 'navigator'], // 手动选择优先，系统语言回退
      lookupLocalStorage: 'shibei-language',
      caches: ['localStorage'],
    },
  });

export default i18n;
```

```typescript
// src/main.tsx
import './i18n'; // 初始化（副作用导入）
```

### 组件使用

```typescript
// 简单文本
import { useTranslation } from 'react-i18next';

function FolderTree() {
  const { t } = useTranslation('sidebar');
  return <button>{t('newFolder')}</button>;
  // zh: "新建文件夹"  en: "New Folder"
}

// 带插值
function DeleteConfirm({ count }: { count: number }) {
  const { t } = useTranslation('annotation');
  return <p>{t('deleteHighlightConfirm', { count })}</p>;
  // zh: "确定删除此高亮标注？关联的 {{count}} 条评论也会一并删除"
  // en: "Delete this highlight? {{count}} associated comment(s) will also be removed."
}

// 多命名空间
function SyncPage() {
  const { t } = useTranslation(['sync', 'common']);
  return (
    <>
      <h2>{t('sync:title')}</h2>
      <button>{t('common:save')}</button>
    </>
  );
}
```

### 语言切换

```typescript
// 设置页
import { useTranslation } from 'react-i18next';

function AppearancePage() {
  const { i18n, t } = useTranslation('settings');

  const changeLanguage = (lng: string) => {
    i18n.changeLanguage(lng);
    // i18next-browser-languagedetector 自动写 localStorage
  };

  return (
    <select value={i18n.language} onChange={e => changeLanguage(e.target.value)}>
      <option value="zh">中文</option>
      <option value="en">English</option>
    </select>
  );
}
```

### TypeScript 类型安全

```typescript
// src/types/i18next.d.ts
import 'i18next';
import type zhCommon from '../locales/zh/common.json';
import type zhSidebar from '../locales/zh/sidebar.json';
// ... 其他命名空间

declare module 'i18next' {
  interface CustomTypeOptions {
    defaultNS: 'common';
    resources: {
      common: typeof zhCommon;
      sidebar: typeof zhSidebar;
      // ... 其他命名空间
    };
  }
}
```

这样 `t('nonexistentKey')` 会有 TypeScript 编译报错。

## 后端错误消息处理

### 当前现状

Rust 后端通过 `CommandError` 返回中文错误信息，前端直接 toast 显示：

```rust
// 当前
Err(CommandError::InvalidInput("文件夹名称已存在".into()))
```

### 改造方案

后端改为返回错误 code，前端负责翻译：

```rust
// 改造后
Err(CommandError::InvalidInput("error.folderNameExists".into()))
```

```json
// zh/common.json
{
  "error": {
    "folderNameExists": "文件夹名称已存在",
    "deleteFailed": "删除失败",
    "saveFailed": "保存失败"
  }
}
```

前端 toast 层统一做翻译：

```typescript
// 封装 invoke，统一翻译错误
function showError(errorKey: string) {
  const message = i18n.exists(errorKey) ? i18n.t(errorKey) : errorKey;
  toast.error(message);
}
```

**注意**：
- 后端返回的 key 如果在语言包中找不到，降级显示原始 key（防止遗漏翻译导致空白消息）
- 只有手写的 ~15-20 条 `InvalidInput` 验证消息适合改为 i18n key（如"文件夹名称已存在"）
- 系统错误（rusqlite、IO 等 `From<T>` 自动转换的错误）仍为英文原始信息，不做翻译——这些是技术性错误，用户极少遇到，翻译收益低

## Chrome 插件 i18n

Chrome Extension 有自己的 i18n 机制（`chrome.i18n` API），与主应用独立。

### 目录结构

```
extension/
  _locales/
    zh/
      messages.json
    en/
      messages.json
  manifest.json          # 添加 "default_locale": "zh"
```

### 使用

```json
// _locales/zh/messages.json
{
  "popupConnected": { "message": "已连接" },
  "popupDisconnected": { "message": "未连接" },
  "popupSaving": { "message": "正在保存..." },
  "popupSaveSuccess": { "message": "保存成功！" },
  "regionSelectHint": { "message": "选择要保存的区域 — 滚轮或方向键调整层级 — ESC 退出" }
}
```

```javascript
// popup.js
document.getElementById('status').textContent = chrome.i18n.getMessage('popupConnected');
```

插件语言跟随浏览器语言设置，不需要独立的语言切换 UI。

### MAIN world 限制

`region-selector.js` 运行在 MAIN world，无法使用 `chrome.i18n.getMessage()` API。处理方式：

- 注入时由 background.js 读取 `chrome.i18n.getMessage()` 翻译好的字符串，作为参数传入 MAIN world 脚本
- 或者在 `relay.js`（ISOLATED world）中预取翻译文本，通过 `window.postMessage` 传递给 MAIN world

### popup.html 硬编码文案

`popup.html` 模板中有硬编码中文（如"保存到文件夹"、"保存整页"），HTML 模板不支持 `chrome.i18n.getMessage()`。需要在 `popup.js` 初始化时用 JS 替换 DOM 文本内容。

## 实施策略

### Key 命名约定

- 使用 `camelCase` key：`newFolder`、`deleteConfirm`、`saveSuccess`
- 嵌套按语义分组：`error.saveFailed`、`dialog.deleteTitle`
- 通用词放 `common` 命名空间：`save`、`cancel`、`delete`、`confirm`、`edit`
- 专属词放对应命名空间：`sync.testConnection`、`encryption.setPassword`

### 迁移顺序

按功能模块逐步迁移，每步可独立编译运行：

1. **基础设施** — 安装依赖、配置 i18n.ts、类型声明、语言切换 UI
2. **common 命名空间** — 抽取通用文案（按钮、确认弹窗、通用错误），影响面最广但单个改动最小
3. **sidebar + search** — 侧栏导航、搜索框
4. **annotation + reader** — 标注面板、阅读器
5. **settings + sync + encryption + lock** — 设置页各子页面（字符串最密集）
6. **后端错误消息** — Rust 改为返回 key，前端翻译层
7. **Chrome 插件** — 独立迁移，使用 chrome.i18n API
8. **英文翻译** — 完成所有 en/*.json

### 附带修复

迁移过程中顺带处理以下问题：

- **`alert()` 调用**：FolderTree/FolderEditDialog 中的 7 处 `alert()` 改为 toast 或 Modal，使其可翻译
- **`toLocaleString("zh-CN")` 硬编码**：SyncPage.tsx 中日期格式化改为跟随 `i18n.language`（如 `toLocaleString(i18n.language)`），其他 `toLocaleDateString()` 调用同理
- **Hook 中的翻译**：`useSync`、`useFolders` 等 hooks 中的 toast 消息同样使用 `useTranslation()` hook 获取 `t` 函数

### 不在 v1.8 范围

- RTL（从右到左）语言支持
- annotator.js 内的文案（标注脚本在 iframe 中运行，确认无用户可见字符串，不需要 i18n）
- 超过中英文之外的其他语言
- 语言包远程加载 / 热更新
- 自动翻译工具集成

## 风险与注意事项

1. **字符串遗漏**：迁移 400 条字符串容易遗漏。每步迁移完后通 `grep -r` 检查中文硬编码残留
2. **key 不存在降级**：i18next 的 `fallbackLng: 'zh'` 确保英文缺翻译时显示中文，不会出现空白
3. **插值安全**：`escapeValue: false` 是安全的（React 自动转义），但语言包中不要包含 HTML 标签
4. **测试兼容**：现有前端测试需要在 test setup 中初始化 i18n（mock 或真实），否则 `useTranslation` 报错
5. **annotator.js 独立**：标注脚本运行在 iframe 中，无法访问主应用的 i18next 实例，暂不纳入 i18n
