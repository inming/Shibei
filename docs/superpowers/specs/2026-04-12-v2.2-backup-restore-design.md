# v2.2 — 本地备份与恢复

## 目标

为未配置云同步的用户提供数据安全网：一键导出完整资料库为 zip 文件，支持从 zip 恢复（覆盖式）。核心场景：灾难恢复（硬盘损坏/系统重装）和设备迁移（换新电脑）。

**不在范围**：单篇导出、增量备份、合并式恢复、备份加密。

---

## 备份文件结构

文件命名：`shibei-backup-{YYYY-MM-DD-HHmmss}.zip`

```
shibei-backup-2026-04-12-153000.zip
├── manifest.json              # 元信息
├── shibei.db                  # SQLite 数据库完整副本
└── storage/                   # 快照文件目录
    ├── {resource_id_1}/
    │   └── snapshot.html
    ├── {resource_id_2}/
    │   └── snapshot.html
    └── ...
```

### manifest.json

```json
{
  "version": 1,
  "app_version": "2.2.0",
  "created_at": "2026-04-12T15:30:00Z",
  "device_id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "resource_count": 42,
  "snapshot_count": 40
}
```

- `version`：格式版本号，用于未来兼容判断。当前恢复只接受 `version: 1`
- `snapshot_count` 可能小于 `resource_count`（云同步场景下部分快照未下载到本地）
- FTS5 虚拟表（`search_index`）不包含在备份中，恢复后重建

## 备份流程

### Tauri Command

```rust
#[tauri::command]
async fn cmd_export_backup(state: State<'_, AppState>, path: String) -> Result<BackupResult, String>
```

`path` 由前端通过 `@tauri-apps/plugin-dialog` 的 `save()` 对话框获取。

### 执行步骤

1. 通过 SQLite `VACUUM INTO` 生成干净的 db 副本到临时文件（不影响运行中的连接池）
2. 统计 resource_count（查 db 副本）和 snapshot_count（遍历 storage 目录）
3. 创建 zip 文件，写入 `manifest.json`
4. 写入 db 副本为 `shibei.db`
5. 遍历 `{base_dir}/storage/` 目录，将所有 `{id}/snapshot.html` 写入 zip 的 `storage/` 下
6. 清理临时 db 文件
7. 返回 `BackupResult { resource_count, snapshot_count, file_size }`

### 错误处理

- 磁盘写入失败 → 清理半成品 zip 文件，返回 `error.backup_write_failed`
- 临时文件创建失败 → 返回 `error.backup_temp_failed`
- 整个过程在异步线程执行，不阻塞 UI

## 恢复流程

### Tauri Command

```rust
#[tauri::command]
async fn cmd_import_backup(state: State<'_, AppState>, app: AppHandle, path: String) -> Result<RestoreResult, String>
```

`path` 由前端通过 `@tauri-apps/plugin-dialog` 的 `open()` 对话框获取（过滤 `.zip`）。

### 执行步骤

1. 打开 zip，读取并校验 `manifest.json`（`version` 必须为 1，必须包含 `shibei.db`）
2. 解压 zip 内容到临时目录 `{base_dir}/restore-tmp/`（先解压再替换，避免解压失败导致数据丢失）
3. 校验临时目录中 `shibei.db` 可正常打开（防损坏文件）
4. 备份当前数据库为 `shibei.db.bak`（防误操作，留一手）
5. 关闭当前数据库连接池
6. 用临时目录中的 `shibei.db` 覆盖当前数据库文件
7. 删除当前 `storage/` 目录，将临时目录中的 `storage/` 移动到位
8. 清理临时目录
9. 重新初始化数据库连接池
10. 触发 FTS 全量重建（`rebuild_all_search_index`）
11. emit 领域事件：`data:resource-changed`、`data:folder-changed`、`data:tag-changed`、`data:annotation-changed`，前端全量刷新
12. 返回 `RestoreResult { resource_count }`

### 连接池动态重建

当前 `AppState` 的 `pool: db::DbPool`（`r2d2::Pool<SqliteConnectionManager>`）不支持运行时替换。

方案：将 `pool` 字段改为 `RwLock<DbPool>`：
- 正常读操作：`state.pool.read().unwrap()` 获取 pool，性能影响可忽略（读锁无竞争）
- 恢复时：`state.pool.write().unwrap()` 获取写锁，替换为新 pool

两个 `AppState`（commands 和 server）都需要改。

### 同步状态处理

恢复时**原样保留** `sync_state` 和 `sync_log`，不主动重置：
- 立刻导出再导入 → 无缝继续同步，零副作用
- 旧备份恢复 → 下次同步时，引擎通过 LWW 合并处理差异（JSONL 断档时已有快照导入回退机制）
- 迁移新机器 → `device_id` 文件不包含在备份中，目标机器保留自己的 device_id

### 安全

- **路径穿越防护**：解压 zip entry 时校验文件路径不包含 `..` 或绝对路径，拒绝恶意 zip
- **恢复前备份**：自动保存 `shibei.db.bak`

## 前端交互

### 设置页新增"数据"分区

侧栏导航在"安全"下方新增"数据"项，对应 `DataPage` 组件。

`SettingsSection` 类型扩展为 `"appearance" | "sync" | "encryption" | "security" | "data"`。

### DataPage 页面布局

```
数据管理
─────────────────────
备份
将资料库完整导出为 zip 文件，包含所有资料和快照。
[创建备份]

─────────────────────
恢复
从备份文件恢复资料库，当前数据将被覆盖。
[从备份恢复]
```

### 创建备份流程

1. 点击"创建备份" → 调用 `save()` 对话框，默认文件名 `shibei-backup-{YYYY-MM-DD-HHmmss}.zip`
2. 用户选路径 → 按钮变为"正在备份..."禁用状态
3. 完成 → toast 提示"备份完成，共 N 个资料"
4. 失败 → toast.error 提示错误

### 恢复流程

1. 点击"从备份恢复" → 调用 `open()` 对话框，过滤 `.zip` 文件
2. 选择文件后弹确认对话框：
   - 基础提示："恢复将覆盖当前所有数据，此操作不可撤销。确认恢复？"
   - 如已配置云同步，额外提示："恢复后下次同步会与云端合并，云端较新的变更（包括删除）将覆盖备份中的旧数据。如需保留备份中的数据，请在恢复后先关闭自动同步。"
3. 确认 → 按钮禁用 + loading 状态
4. 完成 → toast 提示"恢复完成，共 N 个资料"，领域事件触发前端自动刷新
5. 失败 → toast.error

### 判断是否已配置云同步

调用现有命令 `cmd_get_sync_config` 检查是否有 S3 配置。

### 后端错误 key

- `error.backup_write_failed` — 备份文件写入失败
- `error.backup_temp_failed` — 临时文件创建失败
- `error.restore_invalid_format` — 备份文件格式无效
- `error.restore_version_unsupported` — 不支持的备份版本
- `error.restore_db_missing` — 备份中缺少数据库文件
- `error.restore_failed` — 恢复失败

## 跨设备兼容

备份文件完全自包含，天然支持跨设备恢复：
- `shibei.db` 是标准 SQLite 文件，跨平台兼容
- `resources.file_path` 存储相对路径（`storage/{id}/snapshot.html`），不含绝对路径
- `device_id` 不包含在备份中，恢复后使用目标机器自己的 device_id

## 决策摘要

| 决策 | 结论 |
|------|------|
| 范围 | 全量备份 + 覆盖式恢复，不含单篇导出 |
| 格式 | zip（manifest.json + shibei.db + storage/） |
| 恢复策略 | 覆盖式，恢复前自动保留 shibei.db.bak |
| 同步处理 | 原样保留 sync_state/sync_log，恢复时提示同步用户 |
| 连接池 | `DbPool` → `RwLock<DbPool>` 支持运行时替换 |
| FTS 索引 | 不包含在备份中，恢复后重建 |
| UI 入口 | 设置页新增"数据"分区 |
| 跨设备 | 天然支持 |
