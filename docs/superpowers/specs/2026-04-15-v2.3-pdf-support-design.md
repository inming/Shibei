# v2.3 — PDF 支持设计文档

## 目标

让拾贝支持 PDF 资料的保存、阅读和标注，与现有 HTML 快照体验对齐。

## 需求总结

- **来源**：浏览器插件保存 + 本地文件导入
- **阅读**：PDF.js 渲染，虚拟滚动
- **标注**：文本高亮 + 评论，复用现有 highlights/comments 表
- **搜索**：Rust 端提取纯文本，FTS5 自动索引
- **同步**：复用 S3 快照同步管道

## 架构方案

**最小侵入**：PDF 作为新的 `resource_type` 平行于 webpage，复用现有存储/同步/搜索/标注管道，只在内容渲染层分叉。

- ReaderView 根据 resource_type 条件渲染 — HTML 走 iframe，PDF 走 `PDFReader` 子组件
- 外围 UI（meta 栏、进度条、AnnotationPanel、面板折叠）全部复用
- 后端数据模型、同步协议、MCP、备份恢复零改动

---

## 1. 存储与数据模型

### 文件存储

PDF 保存为 `storage/{resource_id}/snapshot.pdf`，与 HTML 的 `snapshot.html` 并列。`storage::save_snapshot()` 扩展为接受文件扩展名参数。

### DB 变更

无 migration。`resource_type` 字段已存在（migration 001，默认 `'webpage'`），新增值 `"pdf"`。`file_path` 存相对路径，PDF 为 `storage/{id}/snapshot.pdf`。

### 纯文本提取

新增 `src-tauri/src/pdf_text.rs` 模块，使用 `pdf-extract` crate。保存时调用，写入 `plain_text` 字段。提取失败不阻塞保存（best-effort，与 HTML 一致）。FTS 索引自动生效，无需改搜索模块。

### 同步兼容

`snapshot.pdf` 与 `snapshot.html` 走同一套 S3 上传/下载逻辑。S3 key 格式：`snapshots/{resource_id}.pdf.gz`（对应 HTML 的 `.html.gz`）。sync engine 根据 resource_type 决定扩展名。

---

## 2. 内容获取

PDF 不走 `shibei://` 协议（PDF.js 需要 ArrayBuffer，不是让 iframe 直接加载）。协议处理器不改动。

新增 Tauri command：

```
cmd_read_pdf_bytes(resource_id: String) -> Vec<u8>
```

读取 `snapshot.pdf` 返回二进制。前端 PDFReader 调用此命令获取数据，传给 PDF.js 的 `getDocument({ data })` 渲染。

---

## 3. 前端渲染

### ReaderView 条件分支

`ReaderView.tsx` 根据 `resource.resource_type` 切换内容区：

- `"webpage"` / `"html"` / `"html_fragment"` → 现有 iframe + annotator.js 逻辑不变
- `"pdf"` → 渲染 `PDFReader` 子组件

外围 UI 全部复用：meta 栏 auto-hide、阅读进度条、AnnotationPanel、面板折叠。

### PDFReader 组件

新增 `src/components/PDFReader.tsx`：

- 调用 `cmd_read_pdf_bytes` 获取 PDF 数据
- `pdfjs-dist` 的 `getDocument({ data })` 解析文档
- 逐页渲染：每页一个 canvas（渲染层）+ 一个 div（文本层，PDF.js `renderTextLayer`）
- **虚拟滚动**：只渲染可视区域 ±1 页的 canvas，其余用高度占位 div，避免大 PDF 内存爆炸
- 滚动事件向 ReaderView 传递 `scrollPercent` 和 `direction`，复用进度条和 meta 栏 auto-hide

### 文本选择

PDF.js 文本层是透明 div 覆盖在 canvas 上，浏览器原生选择即可。选中后弹出工具条（复用现有选区工具条样式），创建高亮。

### PDF Worker

`pdfjs-dist` 需要 web worker 解析 PDF。worker 文件放在 `public/pdf.worker.min.mjs`，通过 `GlobalWorkerOptions.workerSrc` 配置。

## 4. PDF 标注系统

### Anchor 格式

现有 HTML 高亮 anchor 为 `{ textPosition, textQuote }` 结构。PDF 新增 anchor 类型：

```json
{
  "type": "pdf",
  "page": 3,
  "charIndex": 120,
  "length": 45,
  "textQuote": {
    "exact": "选中的文本内容",
    "prefix": "前10字符",
    "suffix": "后10字符"
  }
}
```

- `page` + `charIndex` + `length`：精确定位（PDF.js `getTextContent()` 字符流中的偏移）
- `textQuote`：模糊回退，PDF 重新排版导致偏移变化时用文本匹配恢复

### 后端零改动

`anchor` 字段在 DB 中是 TEXT（JSON 字符串），不关心内部结构。highlights/comments 的 CRUD、同步、搜索索引全部透传。

### 标注流程

1. **创建高亮**：用户选中文本 → 获取 Selection range → 计算 page/charIndex/length → 构建 anchor JSON → 调用 `cmd_create_highlight`（已有命令）
2. **渲染高亮**：页面渲染后，遍历当前页高亮列表 → 根据 anchor 定位文本层中的 span → 叠加半透明背景色（与 HTML 高亮同一色板）
3. **点击高亮**：点击高亮区域 → 展开 AnnotationPanel 对应评论 → 双向联动

### 与 AnnotationPanel 通信

不走 postMessage（iframe 专用）。PDFReader 作为 React 子组件，通过 props/callbacks 与 ReaderView 通信。AnnotationPanel 的高亮列表、评论编辑、点击跳转等 UI 全部复用。

## 5. Chrome 插件 PDF 保存

### 检测逻辑

popup.js 打开时检查当前 tab：

- URL 以 `.pdf` 结尾，或
- 通过 content script 检测 `document.contentType === "application/pdf"`

检测到 PDF 后，插件面板切换为 PDF 保存模式 — 隐藏"选区保存"按钮（PDF 不支持选区），保存按钮文案不变。

### 下载与上传

1. Background service worker 中 `fetch(tab.url)` 下载原始 PDF 二进制（自动携带 cookie）
2. POST 到 `http://127.0.0.1:21519/api/save-raw`，`Content-Type: application/pdf`，body 为原始字节
3. 元数据通过 `X-Shibei-*` headers 传递（title 取 tab.title 或文件名，URL 取 tab.url）

### Server 端

`handle_save_raw()` 根据 Content-Type 分支：

- `text/html` → 现有逻辑（`snapshot.html` + HTML 纯文本提取）
- `application/pdf` → `resource_type = "pdf"`，保存 `snapshot.pdf`，调用 `pdf_text::extract_plain_text`

## 6. 本地文件导入

新增 Tauri command：

```
cmd_import_pdf(file_path: String, folder_id: Option<String>) -> Resource
```

流程：读取文件 → 生成 resource_id → 保存 snapshot.pdf → 提取纯文本 → 写入 DB → emit `data:resource-changed` 事件。

前端入口：资料列表顶部操作区增加"导入 PDF"按钮（与搜索栏同行），用 `@tauri-apps/plugin-dialog` 的 `open()` 选择文件（filter: `*.pdf`）。导入到当前选中的文件夹（未选中则导入到根目录）。

---

## 7. 边界情况与限制

### 加密 PDF

`pdf-extract` 无法提取加密 PDF 文本。保存不阻塞，`plain_text` 为空，搜索只能匹配标题/URL/标注。PDF.js 渲染时遇到密码保护弹出密码输入框（PDF.js 内置）。

### 超大 PDF

虚拟滚动只渲染可视区域 ±1 页，内存可控。纯文本提取在 `tokio::spawn_blocking` 后台线程执行，不阻塞 UI。

### 扫描版 PDF（图片 PDF）

文本层为空，无法高亮文本，也无法提取纯文本。可正常阅读、添加资料级笔记，但不支持文本高亮。

### S3 同步

PDF 文件通常比 HTML 大。现有按需下载机制天然适用 — 元数据先同步，PDF 文件打开时才下载。gzip 对 PDF 压缩效果有限（PDF 内部已压缩），但统一走 `.pdf.gz` 保持一致性。

### 不在本期范围

- OCR 支持（扫描版 PDF 文本识别）
- PDF 内链接跳转
- PDF 表单填写
- PDF 页面缩放（固定宽度适配阅读器区域）
- 导出带标注的 PDF
- 密钥轮换
