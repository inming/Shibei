# v1.2 — 体验深度打磨 设计文档

## 概述

v1.2 的目标是把拾贝从"能用"推向"好用"。核心功能模块：标签系统完善、拖拽与移动、资料列表增强、视觉打磨。

### 设计决策汇总

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 拖拽实现 | `@dnd-kit/core` + `@dnd-kit/sortable` | 提供拖拽预览、碰撞检测、排序动画、无障碍支持，~20KB gzip |
| 批量操作 | Finder 式多选（Cmd+Click / Shift+Click）+ 右键菜单 | 资料列表 340px 宽，checkbox 挤占空间；桌面用户最自然 |
| 标签筛选逻辑 | OR（并集） | 个人资料库场景标签更多是分类标记，OR 覆盖绝大多数需求 |
| 资料排序 | 创建时间 / 抓取时间切换，升降序 | 短期够用，接口预留扩展性 |
| 技术债 | 移至 v1.2.1 | 聚焦功能交付，单用户桌面应用连接池非瓶颈 |

## 一、后端补全

### 1.1 暴露已有 DB 函数为 Tauri Command

以下 DB 函数已实现但未暴露为 Tauri Command，需在 `commands/mod.rs` 中添加薄封装：

| 命令 | DB 函数 | 用途 |
|------|---------|------|
| `cmd_add_tag_to_resource(resource_id, tag_id)` | `db::tags::add_tag_to_resource` | 给资料打标签 |
| `cmd_remove_tag_from_resource(resource_id, tag_id)` | `db::tags::remove_tag_from_resource` | 移除标签 |
| `cmd_update_tag(id, name, color)` | `db::tags::update_tag` | 编辑标签名/颜色 |
| `cmd_get_resources_by_tag(tag_id)` | `db::tags::get_resources_by_tag` | 标签筛选 |
| `cmd_reorder_folder(id, new_sort_order)` | `db::folders::reorder_folder` | 文件夹排序 |
| `cmd_move_resource(id, new_folder_id)` | `db::resources::move_resource` | 资料移动 |

### 1.2 新增 DB 函数 + Command

**`update_resource(id, title, description)`**

更新资料元信息。只开放 title 和 description 两个字段，URL/域名等由抓取决定不可编辑。

```sql
UPDATE resources SET title = ?1, description = ?2 WHERE id = ?3
```

**`list_resources_by_folder` 排序参数扩展**

当前硬编码 `ORDER BY created_at DESC`，改为接受参数：
- `sort_by`: `created_at`（默认）| `captured_at`
- `sort_order`: `desc`（默认）| `asc`

由于 SQL 的 ORDER BY 列名不能参数化绑定，在 Rust 代码中用枚举匹配构造 SQL 字符串（白名单方式，避免注入）。

### 1.3 前端 invoke 封装

在 `src/lib/commands.ts` 中同步添加 TypeScript 封装函数，与后端 command 一一对应。

## 二、标签系统

### 2.1 标签管理 UI

在 Sidebar 的 TagFilter 区域增强：

- **创建标签**：TagFilter 顶部加 "+" 按钮，点击弹出 inline popover（不用 Modal），输入名称 + 选颜色（预设 8-10 种颜色圆点供选择）。
- **编辑标签**：右键标签 → 上下文菜单 → "编辑" → 同样的 popover，可改名/换色。
- **删除标签**：右键菜单 → "删除" → 确认 Modal（提示"将从 N 个资料中移除"）。

### 2.2 标签筛选

- 点击 TagFilter 中的标签切换选中/未选中状态（高亮背景色）。
- 选中标签时，ResourceList 过滤为拥有任一选中标签的资料（OR 逻辑）。
- 无标签选中时显示当前文件夹全部资料。
- 标签筛选叠加在文件夹上：先按 folder 过滤，再按 tag 过滤。

### 2.3 资料打标签

- ResourceList 中资料项右键菜单增加"标签"子菜单，列出所有标签，已关联的打勾，点击切换。
- 批量选中多个资料时，右键菜单的标签操作作用于所有选中项。

### 2.4 数据流

```
TagFilter (选中 tag_ids)
  → LibraryView 状态提升
  → ResourceList 接收 selectedTagIds
  → 前端过滤（tag_ids 为空时不过滤）
```

标签筛选用前端过滤——资料量是个人级别（百~千级），前端过滤响应更快，避免每次切换标签都走 IPC。需要先通过 `getTagsForResource` 获取每个资料的标签，缓存在 `useResources` 中。

## 三、拖拽与移动

### 3.1 依赖

`@dnd-kit/core` + `@dnd-kit/sortable`，同一生态，~20KB gzip。

### 3.2 DnD 上下文结构

```
<DndContext> (LibraryView 级，统一管理跨区域拖拽)
  ├── FolderTree (Droppable + SortableContext 容器)
  │   └── FolderItem (Draggable + Droppable)
  └── ResourceList
      └── ResourceItem (Draggable)
```

一个 `DndContext` 放在 LibraryView 层级，因为资料拖到文件夹需要跨两个组件区域。

### 3.3 文件夹拖拽排序

- 同级文件夹支持拖拽排序。
- 拖拽时显示插入指示线（drop indicator），放置后对同级文件夹按新顺序逐个调用 `cmd_reorder_folder(id, new_sort_order)` 更新 sort_order。若性能成为问题，后续可新增批量接口。
- 只允许同级排序，不通过拖拽排序来移动层级。

### 3.4 文件夹层级移动

- 拖拽文件夹到另一个文件夹上悬停 > 300ms 时，目标文件夹高亮表示"放入"。
- 放置后调用 `cmd_move_folder(id, new_parent_id)`。
- 后端已有循环检测，前端依赖后端校验，失败时 toast 提示。

### 3.5 资料跨文件夹移动

- ResourceList 中资料项可拖拽到左侧 FolderTree 的文件夹节点上。
- 目标文件夹高亮，放置后调用 `cmd_move_resource(id, new_folder_id)`。
- 多选状态下拖拽任一选中项，移动全部选中资料。

### 3.6 拖拽视觉反馈

- 拖拽中：原位置保留半透明占位，`DragOverlay` 组件显示拖拽预览。
- 合法目标：蓝色边框高亮。
- 非法目标（如拖文件夹到自身子节点）：不高亮，光标显示禁止。

## 四、资料列表增强

### 4.1 排序

- ResourceList 顶部增加排序控件：下拉选择（创建时间 / 抓取时间）+ 升降序切换按钮（箭头图标）。
- 默认：创建时间降序（与当前行为一致）。
- 排序通过后端 `cmd_list_resources` 的新参数实现。

### 4.2 Finder 式多选

- **单击**：选中单个资料（清除其他选中），同时更新预览面板。
- **Cmd+Click**：切换该项选中状态，不影响其他已选项。
- **Shift+Click**：从上次点击位置到当前位置范围选中。
- 选中状态用蓝色背景标识，多选时多个项目同时高亮。
- 选中计数显示在列表顶部排序控件旁，如"已选 3 项"。

### 4.3 批量操作

多选状态下右键弹出批量上下文菜单：

- **删除** — 确认 Modal（"确认删除 N 个资料？"）
- **移动到...** — 子菜单展示文件夹树，点击目标文件夹执行移动
- **标签** — 子菜单列出所有标签，点击为所有选中资料添加/移除

### 4.4 资料元信息编辑

- 单选右键菜单增加"编辑"选项。
- 点击弹出 Modal，可编辑标题和描述。
- 保存调用 `cmd_update_resource(id, title, description)`。

### 4.5 右键菜单汇总

单选右键：

```
编辑
标签 →  [标签列表，已关联打勾]
移动到 →  [文件夹树]
──────
删除
```

多选右键：

```
标签 →  [标签列表]
移动到 →  [文件夹树]
──────
删除 (N 项)
```

## 五、视觉打磨

### 5.1 选区工具条边界检测

ReaderView 中 SelectionToolbar 当前固定在选区上方。改为：
- 选区靠近顶部时，工具条显示在选区下方。
- 靠近左右边缘时水平偏移，确保不超出 iframe 可视区域。
- 计算逻辑在 annotator.ts 的定位函数中实现。

### 5.2 外链处理

iframe 内链接默认不可点击（拦截 click 不做任何处理），避免干扰标注操作。按住 Ctrl 时点击才在外部浏览器打开，并显示 toast"已在浏览器中打开"（2 秒自动消失，用已有的 `react-hot-toast`）。

### 5.3 删除按钮过渡动画

ResourceList 中 hover 显示的删除按钮加 `opacity` + `transition` 过渡（CSS 改动）。

### 5.4 骨架屏

ResourceList 和 PreviewPanel 加载时显示骨架占位（灰色矩形脉冲动画），替代当前的 Spinner。纯 CSS 实现（`@keyframes` 渐变动画），不引入库。
- ResourceList 骨架：3-4 个资料项占位条。
- PreviewPanel 骨架：标题条 + 内容块占位。

### 5.5 键盘导航

- **资料列表**：`↑`/`↓` 切换选中项，`Enter` 打开阅读器，`Delete`/`Backspace` 删除（需确认）。
- **文件夹树**：`↑`/`↓` 切换文件夹，`→` 展开，`←` 折叠，`Enter` 选中。
- **焦点区域**：`Tab` 在 FolderTree → ResourceList → PreviewPanel 间切换。

### 5.6 无障碍

- 文件夹树和资料列表加 `role="tree"` / `role="listbox"` + `aria-selected`。
- 上下文菜单加 `role="menu"` + `aria-label`。
- Modal 和 Popover 加焦点陷阱（focus trap）。
- 可拖拽元素的无障碍由 `@dnd-kit` 内置支持。

## 实现顺序

1. **后端补全** — 暴露缺失的 Tauri Command + 新增 update_resource + 排序参数
2. **标签系统** — 管理 UI + 筛选 + 资料打标签
3. **拖拽与移动** — @dnd-kit 集成 + 文件夹排序/移动 + 资料移动
4. **资料列表增强** — 排序 + Finder 式多选 + 批量操作 + 元信息编辑
5. **视觉打磨** — 工具条边界 + 外链处理 + 动画 + 骨架屏 + 键盘导航 + 无障碍

各模块相对独立，但后端补全是所有前端功能的前置条件。
